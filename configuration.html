Configuration Guide

Enterprise configuration guide for Raghim AI Enterprise Agent Platform

Configuration Flow: Set required env vars -> tune provider/model settings -> enforce security controls -> validate with health checks.

Configuration Topics

  • Environment Variables
  • Database Configuration
  • AI Model Settings
  • Security Configuration
  • Performance Tuning

Environment Variables

Configure your system using environment variables.

License Note: The platform license is embedded in the compiled image. No license key environment variable is required.
Variable Description Default
JWT_SECRET_KEY JWT secret for authentication Generated
POSTGRES_PASSWORD PostgreSQL database password Generated
REDIS_PASSWORD Redis cache password Generated
GPU_ENABLED Enable GPU acceleration true
LLM_PROVIDER LLM backend: vllm (on-premise, default) or openai (cloud — document content leaves your infrastructure) vllm
VLLM_BASE_URL URL of your vLLM inference server (required when LLM_PROVIDER=vllm) http://host.docker.internal:8000
VLLM_MODEL Model name served by your vLLM instance mistralai/Mistral-Small-Instruct-2409
OPENAI_API_KEY Required when LLM_PROVIDER=openai. Note: queries and document content are sent to OpenAI servers. —
EMBEDDING_MODEL Sentence-transformer model for embeddings (bundled in image) BAAI/bge-m3
ALLOWED_ORIGINS Comma-separated CORS allowed origins. If blank, uses FRONTEND_URL and API_URL defaults. Empty
WIDGET_DOMAINS Additional domains allowed to embed chat widgets Empty
SSO_ENABLED Enable Single Sign-On via OpenID Connect (OIDC) false
ENABLE_API_DOCS Expose OpenAPI docs at /docs and /redoc. Disable in production. true
RUN_MIGRATIONS_ON_STARTUP Run database migrations on API startup. Set false on secondary replicas. true
AUTH_COOKIE_SECURE Set Secure flag on auth cookie. Must be true in production (HTTPS). true
FRONTEND_URL Frontend access URL https://<app-domain>
API_URL Backend API URL https://<api-domain>

Database Configuration

Configure database settings for optimal performance.

PostgreSQL Settings

# Database configuration
POSTGRES_DB=raghim_ai
POSTGRES_USER=raghim_user
POSTGRES_PASSWORD=your-secure-password
POSTGRES_HOST=postgres
POSTGRES_PORT=5432

# Or use a full connection URL (takes precedence)
DATABASE_URL=postgresql://raghim_user:your-secure-password@postgres:5432/raghim_ai

# Connection pooling (per API replica)
DB_POOL_SIZE=150
DB_MAX_OVERFLOW=150
DB_POOL_TIMEOUT=10
DB_STATEMENT_TIMEOUT_MS=30000

Redis Cache Settings

# Redis configuration
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=secure_password
REDIS_DB=0

# Cache settings
CACHE_TTL=3600
SESSION_TTL=86400

AI Model Settings

Configure AI models and processing parameters.

Model Configuration

# LLM provider (vllm = on-premise | openai = cloud)
LLM_PROVIDER=vllm

# vLLM — self-hosted inference (data stays on your infrastructure)
VLLM_BASE_URL=http://host.docker.internal:8000
VLLM_MODEL=mistralai/Mistral-Small-Instruct-2409

# OpenAI — cloud inference (document content sent to OpenAI servers)
# LLM_PROVIDER=openai
# OPENAI_API_KEY=your-api-key

# Embeddings (sentence-transformer model bundled in image)
EMBEDDING_MODEL=BAAI/bge-m3

Model Token Settings

# RAG token budget (when LLM_PROVIDER=openai)
OPENAI_TIER_HYBRID_TOKENS=<profile-default>
OPENAI_RAG_REASONING_EFFORT=<policy-default>

Security Configuration

Configure security settings and access controls.

Authentication Settings

# JWT configuration (only the secret key is operator-configurable)
JWT_SECRET_KEY=your-secret-key-minimum-32-chars

# SSO via OpenID Connect
SSO_ENABLED=false
SSO_OIDC_DISCOVERY_URL=https://idp.example.com/.well-known/openid-configuration
SSO_CLIENT_ID=your-client-id
SSO_CLIENT_SECRET=your-client-secret
SSO_PROVIDER_NAME=SSO

CORS & Cookie Configuration

IP filtering and rate limiting are handled at the infrastructure/network layer (reverse proxy, firewall) rather than as application environment variables.

# CORS — restrict to your actual domains
ALLOWED_ORIGINS=https://your-app-domain.com,https://your-api-domain.com

# Widget embedding — only list domains that host your embedded chat widgets
WIDGET_DOMAINS=https://your-customer-site.com

# Auth cookie — must be true when serving over HTTPS
AUTH_COOKIE_SECURE=true

Performance Tuning

Optimize system performance for your environment.

Resource Allocation (Docker Compose)

These set Docker Compose resource constraints for each service container.

# Docker Compose resource limits for the backend container
BACKEND_CPU_LIMIT=8.0
BACKEND_MEMORY_LIMIT=8G
BACKEND_CPU_RESERVATION=2.0
BACKEND_MEMORY_RESERVATION=4G

# Celery worker concurrency (document processing)
CELERY_CONCURRENCY=4

Optimization Settings

# API documentation (disable in production)
ENABLE_API_DOCS=false

# Database migration on startup (disable on secondary API replicas)
RUN_MIGRATIONS_ON_STARTUP=true

# Scheduled background jobs (disable on secondary API replicas)
ENABLE_APSCHEDULER=true

# Logging
LOG_LEVEL=INFO
