Authentication

All API endpoints require authentication except for login and registration.

POST /api/auth/login

Authenticate user and receive JWT token

Request Body:

{
  "username": "string",
  "password": "string"
}

Response:

{
  "access_token": "string",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "username": "string",
    "email": "string",
    "tier": "FREE"
  }
}
POST /api/auth/register

Register new user account

Request Body:

{
  "username": "string",
  "email": "string",
  "password": "string",
  "full_name": "string"
}
GET /api/auth/me

Get current user information

Headers:

Authorization: Bearer <token>

Document Management

Upload, process, and manage documents in collections.

POST /api/documents/upload

Upload documents for processing

Request:

Content-Type: multipart/form-data

FormData:
- files: File[]
- collection_name: string (optional)
- chunk_strategy: string (optional)

Response:

{
  "task_id": "string",
  "status": "processing",
  "message": "Documents uploaded successfully"
}
GET /api/documents/collections

List all document collections

Response:

[
  {
    "id": 1,
    "name": "string",
    "description": "string",
    "document_count": 10,
    "created_at": "2024-01-01T00:00:00Z"
  }
]
POST /api/documents/search

Search documents using vector similarity

Request Body:

{
  "query": "string",
  "collection_name": "string",
  "limit": 10,
  "threshold": 0.7
}

Chat & AI

Interact with AI models for question-answering and document analysis.

POST /api/chat

Send message to AI chat interface

Request Body:

{
  "message": "string",
  "prompt_id": "string (optional)",
  "collection_name": "string (optional)",
  "stream": true
}

Response (Streaming):

data: {"type": "content", "content": "partial response"}
data: {"type": "sources", "sources": [...]}
data: {"type": "done", "message": "complete"}
GET /api/chat/history

Get chat conversation history

Query Parameters:

?conversation_id=string&limit=50

Live Chat System

Real-time chat with agent handoff capabilities.

POST /api/live-chat/handoff

Request handoff to human agent

Request Body:

{
  "conversation_id": "string",
  "reason": "string",
  "severity": "low|medium|high|critical"
}
GET /api/live-chat/sessions

Get active chat sessions

Response:

[
  {
    "conversation_id": "string",
    "status": "active|queued|handoff_requested",
    "user_id": 1,
    "created_at": "2024-01-01T00:00:00Z"
  }
]
POST /api/live-chat/accept

Accept handoff request

Request Body:

{
  "conversation_id": "string"
}

Banking Integration

Specialized endpoints for financial document processing.

POST /api/banking/process

Process banking documents with encryption

Request Body:

{
  "query": "string",
  "document_type": "statement|transaction|report",
  "encryption_key": "string (optional)"
}
POST /api/banking/upload

Upload encrypted banking documents

Request:

Content-Type: multipart/form-data

FormData:
- files: File[]
- encryption_key: string
- document_type: string

POST /api/admin/users

GET /api/admin/metrics

Analytics

Widgets

Widget management and embedding endpoints.

POST /api/widgets/create

Create new chat widget

Request Body:

{
  "name": "string",
  "description": "string",
  "collection_id": 1,
  "theme": "light|dark",
  "position": "bottom-right|bottom-left"
}
GET /api/widgets/embed/{widget_id}

Get widget embed code

Error Codes

403 Forbidden Insufficient permissions
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Server error occurred
