Deployment Best Practices

Follow these guidelines for optimal deployment and configuration.

Environment Setup

  • Use dedicated servers for production
  • Implement proper network segmentation
  • Configure firewall rules appropriately
  • Use SSL/TLS certificates

Resource Planning

  • Allocate sufficient RAM (16GB minimum, 32GB+ recommended for offline LLM inference)
  • Plan for GPU acceleration if available
  • Ensure adequate storage space
  • Monitor resource utilization

Backup Strategy

  • Regular database backups
  • Document storage backups
  • Configuration file backups
  • Test restore procedures

Update Management

  • Schedule regular updates
  • Test updates in staging environment
  • Maintain rollback procedures
  • Document update procedures

Optional Services Management

  • Enable the monitoring stack only when your host has sufficient resources (adds ~2–4 GB RAM)
  • Review connector service logs regularly for integration errors
  • Disable unused connector integrations to reduce the attack surface
  • Use Docker Compose profile flags to control which optional services start on boot

Security Best Practices

Implement comprehensive security measures for your deployment.

Authentication & Authorization

  • Strong Passwords: Enforce complex password policies
  • Multi-Factor Authentication: Enable MFA for all users
  • Role-Based Access: Implement least privilege access
  • Session Management: Configure appropriate session timeouts

Data Protection

  • Encryption: Enable encryption at rest and in transit
  • Access Controls: Implement IP whitelisting
  • Data Classification: Classify and protect sensitive data
  • Retention Policies: Implement data retention policies

Network Security

  • HTTPS Only: Enforce HTTPS for all communications
  • Firewall Rules: Configure restrictive firewall rules
  • VPN Access: Use VPN for remote access
  • DDoS Protection: Implement DDoS protection

API & Integration Security

  • Disable API docs in production: Set ENABLE_API_DOCS=false to hide /docs and /redoc from public access
  • Scope CORS precisely: Set ALLOWED_ORIGINS to your exact domain list — never use wildcards in production
  • Widget domain control: Only add trusted domains to WIDGET_DOMAINS; remove domains when embedding is no longer required
  • Connector least-privilege: Grant workflow connectors only the minimum permissions needed for each integration; disable unused connectors
  • Rotate JWT secret periodically: Rotate JWT_SECRET_KEY as part of your credential rotation schedule; all active sessions are invalidated on rotation

Performance Optimization

Optimize system performance for better user experience.

System Optimization

# Performance tuning parameters
LLM_PROVIDER=<approved-provider>
GPU_MEMORY_FRACTION=0.85
WORKER_COUNT=4
THREAD_POOL_SIZE=16
BATCH_SIZE=32
CACHE_TTL=3600
# Disable API documentation in production
ENABLE_API_DOCS=false

Database Optimization

  • Connection Pooling: Configure appropriate pool sizes
  • Indexing: Create proper database indexes
  • Query Optimization: Monitor and optimize slow queries
  • Regular Maintenance: Schedule database maintenance

Caching Strategy

  • Redis Configuration: Optimize Redis settings
  • Cache Policies: Implement appropriate cache TTL
  • Cache Warming: Pre-load frequently accessed data
  • Cache Monitoring: Monitor cache hit rates

Data Management

Best practices for managing documents and data.

Document Organization

  • Use meaningful collection names
  • Implement document tagging
  • Create logical folder structures
  • Regular cleanup of old documents

Data Quality

  • Validate document formats
  • Check OCR accuracy
  • Monitor processing quality
  • Implement quality controls

Storage Management

  • Monitor storage usage
  • Implement compression
  • Archive old data
  • Plan for growth

Backup & Recovery

  • Automated daily backups
  • Test restore procedures
  • Offsite backup storage
  • Disaster recovery planning

Monitoring & Maintenance

Implement comprehensive monitoring and maintenance procedures.

System Monitoring

  • Health Checks: Implement automated health checks
  • Performance Metrics: Monitor CPU, memory, and disk usage
  • Error Tracking: Set up error monitoring and alerting
  • Log Analysis: Regular log review and analysis
  • Worker monitoring: Monitor the rag-worker container alongside core services — document processing failures surface here first: docker logs rag-worker
  • Connector service health: Periodically verify all connector service containers are running; failures here cause agent flow tool calls to fail silently

User Activity Monitoring

  • Access Logs: Monitor user access patterns
  • Usage Analytics: Track system usage metrics
  • Security Events: Monitor for security incidents
  • Compliance Reporting: Generate compliance reports

Maintenance Schedule

# Recommended maintenance schedule
Daily:
- Check system health
- Review error logs
- Monitor resource usage

Weekly:
- Update security patches
- Review user access
- Clean temporary files

Monthly:
- Full system backup
- Performance review
- Security audit
- Update documentation

Quarterly:
- Disaster recovery test
- Capacity planning review
- Security assessment
- Training updates

Alerting Configuration

  • System Alerts: CPU, memory, disk usage thresholds
  • Service Alerts: Service availability monitoring
  • Security Alerts: Failed login attempts, suspicious activity
  • Performance Alerts: Response time thresholds
