Authentication

Secure authentication methods for API access.

JWT Token Authentication

# Login to get token
curl -X POST https://<your-domain>/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "your-username", "password": "your-password"}'

# Use token in subsequent requests
curl -X GET https://<your-domain>/api/documents/collections \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

Integration Verification

Validate your integration in this order:

  • Authenticate and verify token/cookie behavior.
  • Call one read endpoint and one write endpoint.
  • Validate error handling and retry strategy.
  • Confirm audit/trace visibility for key actions.
  • Confirm agent flow triggers and MCP tool calls succeed in staging before enabling in production.

Workflow Automation & MCP Integration

The platform includes a built-in workflow automation layer and Model Context Protocol (MCP) support for connecting external tools and systems.

Workflow Connector Configuration

Workflow connectors are configured through the platform's Admin → Integrations interface. Once configured, connectors can:

  • Read from and write to the RAG knowledge base
  • Trigger agent flows on schedule or on external events
  • Route flow outputs to external services and APIs
  • Synchronise data between external systems and document collections

MCP Tool Integration

The platform supports the Model Context Protocol (MCP) for connecting external tools, databases, and APIs to agent flows at runtime.

MCP Server Configuration

Add MCP servers via Admin → MCP Tools. Each server exposes a set of callable tools available within flow nodes.

SQL Database Connector

Connect SQL databases (PostgreSQL, MySQL, and others) to enable natural-language-to-SQL query flows within the agent engine.

Tool Allowlisting

Each MCP server's tools can be individually allowlisted to enforce least-privilege access within flows.

Agent Flow Triggers

Agent flows can be triggered via the REST API, on a schedule, or by events within the connector layer. Flows support conditional branching, human approval steps, and multi-step tool execution.

# Trigger an agent flow via API (illustrative)
POST /api/agent/flows/<flow-id>/run
Authorization: Bearer YOUR_JWT_TOKEN
Content-Type: application/json

{
  "note": "Trigger payload schema available in approved technical onboarding"
}

Code Examples

Practical examples for common integration scenarios.

Note: The code examples below are illustrative pseudocode demonstrating integration patterns. No official SDK is published. Implement using your preferred HTTP client library (e.g. fetch, axios, requests).

JavaScript Integration

// Initialize Raghim AI client
const raghimAI = new RaghimAIClient({
  apiUrl: 'https://<your-domain>/api',
  token: 'your-jwt-token'
});

// Upload document
const uploadResult = await raghimAI.documents.upload(file);

// Send query
const response = await raghimAI.chat.send({
  message: 'What is this document about?',
  collectionId: uploadResult.collection_id
});

Python Integration

import requests

# Initialize client
api_url = 'https://<your-domain>/api'
headers = {'Authorization': 'Bearer your-jwt-token'}

# Upload document
with open('document.pdf', 'rb') as f:
    files = {'file': f}
    response = requests.post(f'{api_url}/documents/upload', 
                           files=files, headers=headers)

# Send query
query_data = {
    'message': 'Summarize this document',
    'collection_id': response.json()['collection_id']
}
response = requests.post(f'{api_url}/chat', 
                        json=query_data, headers=headers)
