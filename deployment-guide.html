Prerequisites

System Requirements

  • OS: Linux (Ubuntu 18.04+, CentOS 7+, Alpine 3.10+)
  • RAM: 8GB minimum, 16GB recommended
  • Storage: 20GB+ free disk space
  • CPU: 4 cores minimum, 8 cores recommended
  • GPU: NVIDIA GPU with CUDA support (optional)

Software Requirements

  • Docker: 20.10+
  • Docker Compose: 2.0+
  • Python: 3.7+ (for installer)
  • Git: Latest version

Offline Enterprise Deployment

The offline package provides a complete, self-contained deployment solution with all dependencies included.

1

Download Package

Download the offline enterprise package from your RaghimAI representative.

# Extract the package
tar -xzf rag-enterprise-1.0.26-offline.tar.gz
cd rag-enterprise-1.0.26-offline
2

Run Interactive Installer

Use the interactive installer for guided setup and configuration.

# Launch interactive installer
./INSTALL.sh

# Or run installer directly
./installer/installer.sh
Note: The installer will guide you through license acceptance, environment configuration, and deployment options.
3

Non-Interactive Deployment

For automated deployments or CI/CD pipelines.

# Automated deployment
./DEPLOY.sh
Warning: Non-interactive deployment uses default configuration. Update the .env file for production use.

Cloud Deployment

Deploy on cloud platforms with external GPU support for enhanced performance.

VastAI Deployment

1

Create VastAI Instance

  • Select GPU-enabled instance (RTX 4090 recommended)
  • Choose Ubuntu 20.04+ image
  • Configure SSH access
2

Setup SSH Tunnel

# Setup SSH tunnel for Ollama access
./setup_ssh_env.sh
./setup_ollama_tunnel.sh

RunPod Deployment

1

Create RunPod Instance

  • Select GPU-enabled pod
  • Use RunPod PyTorch template
  • Configure network access
2

Deploy with RunPod Config

# Use RunPod-specific configuration
docker-compose -f runpod-docker-compose.yml up -d

Configuration

Configure your deployment using environment variables and configuration files.

Environment Variables

Variable Description Default
LICENSE_KEY Your enterprise license key Required
JWT_SECRET_KEY JWT secret for authentication Generated
POSTGRES_PASSWORD Database password Generated
REDIS_PASSWORD Redis cache password Generated
GPU_ENABLED Enable GPU acceleration true

Common Issues

Connection Refused

Symptoms: "This site can't be reached" or "ERR_CONNECTION_REFUSED"

Solutions:

  • Check if containers are running: docker ps
  • Verify firewall settings (ports 80, 443)
  • Use external IP for VM deployments
  • Test from server: curl http://localhost

GPU Not Detected

Symptoms: Slow AI responses, GPU not utilized

Solutions:

  • Install NVIDIA Docker runtime
  • Set GPU_ENABLED=true in .env
  • Check CUDA installation
  • Verify GPU availability: nvidia-smi

Memory Issues

Symptoms: Out of memory errors, slow performance

Solutions:

  • Increase system RAM
  • Reduce OLLAMA_MAX_LOADED_MODELS
  • Lower OLLAMA_GPU_MEMORY_FRACTION
  • Use smaller models

Database Connection

Symptoms: Database connection errors

Solutions:

  • Check PostgreSQL container status
  • Verify database credentials
  • Check network connectivity
  • Review database logs
