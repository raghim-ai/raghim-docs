Prerequisites

System Requirements

  • OS: Linux (Ubuntu 18.04+, CentOS 7+, Alpine 3.10+)
  • RAM: 16GB minimum, 32GB+ recommended
  • Storage: 80GB+ free disk space (Docker images ~30–40 GB; model weights ~8–50 GB additional)
  • CPU: 4 cores minimum, 8 cores recommended
  • GPU / VRAM: NVIDIA GPU with CUDA support — required for offline LLM inference (8 GB VRAM minimum for small quantised models; 24 GB+ recommended). GPU may be local or a remote instance accessed via SSH tunnel. Not required if using a managed cloud LLM provider.

Software Requirements

  • Docker: 20.10+
  • Docker Compose: 2.0+
  • Python: 3.7+ (installer scripts only; platform runs inside Docker)
  • Git: Latest version

Offline Enterprise Deployment

The offline package provides a complete, self-contained deployment solution with all dependencies included.

1

Download Package

Download the offline enterprise package from your Raghim AI representative.

# Extract the package
tar -xzf rag-enterprise-2.0.5-offline-managed-local.tar.gz
cd rag-enterprise-2.0.5-offline-managed-local
2

Run Interactive Installer

Use the interactive installer for guided setup and configuration.

# Launch interactive installer
./deploy.sh

# Or run installer directly
./installer/installer.sh
Note: The installer will guide you through license acceptance, environment configuration, and deployment options.
3

Non-Interactive Deployment

For automated deployments or CI/CD pipelines.

# Automated deployment
./deploy.sh
Warning: Non-interactive deployment uses default configuration. Update the .env file for production use.

Optional & Bundled Services

Monitoring Stack

The deployment package includes an optional monitoring stack (Prometheus, Grafana, Sentry) that can be enabled via Docker Compose profiles. These services are disabled by default to conserve resources on smaller deployments.

# Enable monitoring stack (Prometheus + Grafana + Sentry + exporters)
docker compose --profile monitoring up -d

# Check monitoring services
docker ps | grep -E "rag-prometheus|rag-grafana|rag-sentry"
Resource note: The monitoring stack adds approximately 2–4 GB RAM overhead. Ensure your host has sufficient capacity before enabling.

Workflow Connector Services

The platform includes bundled workflow automation and connector services that start automatically alongside the core stack. These power the agent flow, MCP tool execution, and external integration features. No additional configuration is required for basic operation; connector-specific settings are managed through the Admin interface.

# Verify all services including connectors are running
docker compose ps

# Check connector service logs if agent flows fail
docker compose logs --tail=50

Common Issues

Connection Refused

Symptoms: "This site can't be reached" or "ERR_CONNECTION_REFUSED"

Solutions:

  • Check if containers are running:
  • Verify firewall settings (ports 80, 443)
  • Use external IP for VM deployments
  • Test from server:

GPU Not Detected

Symptoms: Slow AI responses, GPU not utilized

Solutions:

  • Install NVIDIA Docker runtime
  • Set LLM_PROVIDER=<approved-provider> in .env
  • Check CUDA installation
  • Verify GPU availability: nvidia-smi

Memory Issues

Symptoms: Out of memory errors, slow performance

Solutions:

  • Increase system RAM
  • Reduce OPENAI_TIER_HYBRID_TOKENS
  • Set OPENAI_RAG_REASONING_EFFORT to none or low
  • Use smaller models

Database Connection

Symptoms: Database connection errors or vector search failures

Solutions:

  • Check PostgreSQL container status: docker logs rag-postgres
  • Verify database credentials in .env
  • Check network connectivity between containers
  • Review database logs: docker ps | grep rag-postgres
  • Note: pgvector is integrated into PostgreSQL — no separate vector database container or port is used
