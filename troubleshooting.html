Connection Issues

Site Can't Be Reached

Symptoms: "This site can't be reached" or "ERR_CONNECTION_REFUSED"

Solutions:

  • Check if containers are running: docker ps
  • Verify firewall settings (ports 80, 443)
  • Use external IP for VM deployments
  • Test from server: curl https://<app-domain>
  • Check Docker binding: ensure containers bind to 0.0.0.0

API Not Responding

Symptoms: Backend API returns timeout or connection errors

Solutions:

  • Check backend container status: docker logs rag-backend
  • Verify database connections
  • Check resource limits (CPU/Memory)
  • Restart backend service
  • Check network connectivity between containers

WebSocket Connection Failed

Symptoms: Live chat or real-time features not working

Solutions:

  • Check WebSocket endpoint: ws://localhost:8000/ws
  • Verify proxy configuration in Nginx
  • Check browser console for WebSocket errors
  • Ensure WebSocket support in browser
  • Check firewall for WebSocket ports

Workflow Connector Service Unavailable

Symptoms: Agent flows using external connectors fail, time out, or return connector errors

Solutions:

  • Verify all connector services are running: docker compose ps
  • Review connector service logs: docker compose logs --tail=100
  • Check connector configuration in Admin → Integrations
  • Verify network connectivity between backend and connector containers
  • Restart connector services if needed: docker compose restart
  • Confirm the backend can reach connector endpoints via internal Docker DNS

Performance Issues

Slow AI Responses

Symptoms: AI responses take 10+ seconds

Solutions:

  • Enable GPU acceleration: LLM_PROVIDER=<approved-provider>
  • Check GPU availability: nvidia-smi
  • Reduce model size or use quantized models
  • Increase OPENAI_RAG_REASONING_EFFORT
  • Pre-load models: ollama pull model-name

High Memory Usage

Symptoms: System running out of memory

Solutions:

  • Reduce OPENAI_TIER_HYBRID_TOKENS
  • Lower OPENAI_RAG_REASONING_EFFORT
  • Use smaller models (3B instead of 7B)
  • Increase system RAM
  • Enable memory cleanup: ./scripts/health_check.sh

Slow Document Processing

Symptoms: Document upload and processing takes too long

Solutions:

  • Check pgvector container status
  • Verify embedding model is loaded
  • Use faster embedding models
  • Check disk I/O performance
  • Optimize chunking strategy

Deployment Issues

Docker Compose Fails

Symptoms: Containers fail to start or exit immediately

Solutions:

  • Check Docker version: docker --version
  • Verify Docker Compose version: docker-compose --version
  • Check available disk space
  • Review container logs: docker-compose logs
  • Check port conflicts

Image Loading Failed

Symptoms: Docker images fail to load from tar.gz files

Solutions:

  • Verify tar.gz file integrity
  • Check available disk space
  • Use docker load -i image.tar.gz manually
  • Check Docker daemon status
  • Restart Docker service if needed

Environment Configuration

Symptoms: Services start but behave unexpectedly

Solutions:

  • Check .env file syntax
  • Verify all required variables are set
  • Check variable values and formats
  • Use interactive installer for proper setup
  • Review environment variable documentation

Document Processing Stalled

Symptoms: Uploaded documents remain in "processing" state indefinitely

Solutions:

  • Check worker container: docker logs rag-worker
  • Verify worker is running: docker ps | grep rag-worker
  • Check Redis connectivity from worker (Celery uses Redis as broker)
  • Restart worker: docker restart rag-worker
  • Verify embedding model availability in worker logs
  • Check available disk space: df -h

Database Issues

PostgreSQL Connection Failed

Symptoms: Database connection errors in logs

Solutions:

  • Check PostgreSQL container: docker logs rag-postgres
  • Verify database credentials in .env
  • Check database initialization
  • Verify network connectivity
  • Check database disk space

Redis Connection Issues

Symptoms: Cache operations failing

Solutions:

  • Check Redis container: docker logs rag-redis
  • Verify Redis password configuration
  • Check Redis memory usage
  • Restart Redis container
  • Check Redis persistence settings

pgvector / Vector Search Issues

Symptoms: Vector search not working or returning no results

Solutions:

  • pgvector runs inside PostgreSQL — check: docker logs rag-postgres
  • Verify PostgreSQL is healthy: docker ps | grep rag-postgres
  • Check pgvector extension is installed: connect to the database and run SELECT * FROM pg_extension WHERE extname = 'vector';
  • Verify embedding model is loaded and accessible
  • Check collection initialisation in the backend logs: docker logs rag-backend
  • Note: there is no separate vector database container — pgvector is part of the PostgreSQL service

GPU Issues

GPU Not Detected

Symptoms: System falls back to CPU processing

Solutions:

  • Install NVIDIA Docker runtime
  • Check GPU availability: nvidia-smi
  • Verify CUDA installation
  • Set LLM_PROVIDER=<approved-provider> in .env
  • Check Docker GPU support: docker run --rm --gpus all nvidia/cuda:11.0-base nvidia-smi

GPU Memory Issues

Symptoms: CUDA out of memory errors

Solutions:

  • Reduce OPENAI_RAG_REASONING_EFFORT
  • Use smaller models
  • Reduce OPENAI_TIER_HYBRID_TOKENS
  • Check GPU memory usage: nvidia-smi
  • Restart containers to clear GPU memory

External GPU Connection

Symptoms: External GPU model provider not accessible

Solutions:

  • Check SSH tunnel status
  • Verify external GPU endpoint
  • Test connection: curl http://external-gpu:11434/api/tags
  • Check firewall settings
  • Verify model provider is running on external GPU

Authentication Issues

Login Failed

Symptoms: Cannot log in with valid credentials

Solutions:

  • Check user account status in database
  • Verify password hashing
  • Check JWT secret configuration
  • Review authentication logs
  • Reset user password if needed

Token Expired

Symptoms: "Token expired" errors

Solutions:

  • Check JWT expiration settings
  • Refresh token or re-login
  • Verify system clock synchronization
  • Check JWT secret key consistency
  • Review token refresh logic

Permission Denied

Symptoms: Access denied for valid users

Solutions:

  • Check user role and permissions
  • Verify organization membership
  • Check resource access policies
  • Review RBAC configuration
  • Update user permissions if needed

SSO Login Failed

Symptoms: OIDC login redirects fail, return errors, or loop back to login page

Solutions:

  • Verify SSO_ENABLED=true is set in .env
  • Check SSO_OIDC_DISCOVERY_URL is reachable from inside the backend container
  • Confirm SSO_CLIENT_ID and SSO_CLIENT_SECRET match your identity provider settings
  • Ensure the redirect URI registered in your IdP matches: <API_URL>/api/sso/callback
  • Review backend logs for OIDC errors: docker logs rag-backend
  • Verify your IdP is accessible from the server network (not blocked by firewall)

Diagnostic Tools

Use these built-in tools to diagnose and resolve issues:

Note: Advanced diagnostic scripts (quick_health_check.sh, database_diagnostic.sh, comprehensive_diagnostic.sh) are included in supported distribution packages. If not present in your package, use the Docker logging and health check commands shown in each section above.

Health Check Script

# Run comprehensive health check
./scripts/health_check.sh

Checks all services, endpoints, and provides status information.

Quick Health Check

# Quick system status
./scripts/quick_health_check.sh

Fast check of critical services and endpoints.

Database Diagnostic

# Database-specific diagnostics
./scripts/database_diagnostic.sh

Comprehensive database health and performance check.

Comprehensive Diagnostic

# Full system diagnostic
./scripts/comprehensive_diagnostic.sh

Enterprise system analysis including logs, resources, and configuration.

Tip: Run diagnostic tools regularly to identify potential issues before they become problems. All tools provide detailed output and recommendations for resolution.
