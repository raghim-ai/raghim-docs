Prerequisites

System Requirements

  • OS: Linux (Ubuntu 18.04+, CentOS 7+, Alpine 3.10+)
  • RAM: 8GB minimum, 16GB recommended
  • Storage: 20GB+ free disk space
  • CPU: 4 cores minimum, 8 cores recommended
  • GPU: NVIDIA GPU with CUDA support (optional)

Software Requirements

  • Docker: 20.10+
  • Docker Compose: 2.0+
  • Python: 3.7+ (for installer)
  • Git: Latest version

Offline Enterprise Deployment

The offline package provides a complete, self-contained deployment solution with all dependencies included.

1

Download Package

Download the offline enterprise package from your RaghimAI representative.

# Extract the package
tar -xzf rag-enterprise-1.0.26-offline.tar.gz
cd rag-enterprise-1.0.26-offline
2

Run Interactive Installer

Use the interactive installer for guided setup and configuration.

# Launch interactive installer
./INSTALL.sh

# Or run installer directly
./installer/installer.sh
Note: The installer will guide you through license acceptance, environment configuration, and deployment options.
3

Non-Interactive Deployment

For automated deployments or CI/CD pipelines.

# Automated deployment
./DEPLOY.sh
Warning: Non-interactive deployment uses default configuration. Update the .env file for production use.

Configuration

Configure your deployment using environment variables and configuration files.

Environment Variables

Variable Description Default
LICENSE_KEY Your enterprise license key Required
JWT_SECRET_KEY JWT secret for authentication Generated
DATABASE_PASSWORD Database password Generated
CACHE_PASSWORD Cache password Generated
GPU_ENABLED Enable GPU acceleration true
FRONTEND_URL Frontend access URL http://localhost
API_URL Backend API URL http://localhost:8000
WIDGET_DOMAINS Widget embedding domains Empty

AI Model Configuration

# AI model settings (configurable)
EMBEDDING_MODEL=multilingual-embedding-model
LLM_MODEL=advanced-language-model
TRANSLATION_MODEL=multilingual-translation-model

# Performance optimization settings
GPU_LAYERS=40
GPU_MEMORY_FRACTION=0.85
NUM_PARALLEL=8
MAX_LOADED_MODELS=3

Deployment Verification

Verify your deployment is working correctly using the provided health checks.

Health Check

# Run comprehensive health check
./scripts/health_check.sh

This will check all services and provide status information.

Service Status

# Check running containers
docker-compose -f docker-compose/docker-compose.yml ps

# View service logs
docker-compose -f docker-compose/docker-compose.yml logs -f

Access URLs

Frontend: http://localhost
Backend API: http://localhost:8000
Health Check: http://localhost:8000/health
Vector Store: http://localhost:8001

Security Considerations

Network Security

  • Configure firewall rules appropriately
  • Use HTTPS in production environments
  • Implement proper CORS policies
  • Consider VPN access for sensitive deployments

Data Protection

  • Encrypt sensitive data at rest
  • Use strong authentication mechanisms
  • Implement proper access controls
  • Regular security updates and patches

Monitoring

  • Enable comprehensive logging
  • Monitor system performance metrics
  • Set up alerting for security events
  • Regular security audits

Compliance

  • Follow industry security standards
  • Implement data retention policies
  • Ensure regulatory compliance
  • Document security procedures
